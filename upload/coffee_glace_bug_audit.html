Coffee Glacé codebase bug and issue audit report

4 critical 6 warnings 5 notes
Security
critical

.env committed to git

.gitignore never excludes .env, so DATABASE_URL is tracked in all 3 commits. Add .env and .env.local to .gitignore immediately and rotate any secrets.

critical

Hardcoded absolute path in DATABASE_URL

Value is file:/home/z/my-project/db/custom.db — a path that only exists on one developer's machine. The app cannot connect to the database on any other system or in production. Should use a relative path like file:./db/custom.db.

critical

db/custom.db committed to git

The actual SQLite database file is tracked in the repository. Any data stored in it is exposed to everyone with repo access. Database files should be in .gitignore.

Build & Configuration
critical

Tailwind content paths don't match project structure

tailwind.config.ts scans ./pages/**, ./components/**, ./app/** — but this project uses a src/ layout. Classes used in src/app and src/components will be purged in production builds. Fix: use ./src/**/*.{js,ts,jsx,tsx}.

warning

typescript.ignoreBuildErrors: true in next.config.ts

All TypeScript errors are silently swallowed during builds. Real type bugs can ship to production unnoticed. This should be removed.

warning

reactStrictMode: false in next.config.ts

Strict Mode is disabled, which suppresses double-render checks in dev that catch side effects, stale closures, and deprecated API usage.

warning

Prisma Post model missing @relation

Post.authorId has no @relation directive and User has no posts field. Prisma won't enforce the foreign key or generate relation query helpers like include: { posts: true }.

warning

Prisma query logging always on (including production)

db.ts always passes log: ['query'] to PrismaClient. All SQL is logged to stdout in production, which degrades performance and can expose sensitive data in logs. Should be conditional on NODE_ENV !== 'production'.

Performance
warning

FloatingParticles duplicated across every section

FloatingParticles (14 Framer Motion animated nodes each) is rendered inside every MenuSection and AboutSection — 6 instances total = 84 simultaneously animating elements just for background decoration. Renders one shared instance instead.

warning

Entire page is "use client"

page.tsx forces all rendering to the client, losing server-side rendering for menu data and static content. The menu data is static and never changes — most sections could be server components.

Data & Content
note

12 duplicate image URLs in menu-data.ts

Multiple products share the exact same Unsplash photo (e.g. double-espresso and turkish-coffee, iced-latte and iced-caramel, all three herbal teas, etc.). Each product should have a unique image.

note

Unused Prisma models and db.ts

The User and Post models and src/lib/db.ts are never imported or used anywhere in the app. Either implement them or remove them to reduce bundle size and confusion.

note

~25 unused dependencies in package.json

Libraries like @dnd-kit, @tanstack/react-query, next-auth, react-hook-form, zustand, recharts, and many others are installed but never used. This bloats build time and bundle size significantly.

note

Geist Mono font referenced but never loaded

globals.css sets --font-mono: var(--font-geist-mono), but layout.tsx only loads Vazirmatn. --font-geist-mono will always be undefined, so monospace text falls back to system fonts.

note

Instagram handle contains invalid character

The contact section shows @coffeeglacé — the accented é is not a valid Instagram handle character (only letters, numbers, . and _ are allowed). This would silently fail if turned into a link.
